# Digital Seva Platform - Deployment Plan

## Information Gathered

### Project Structure
- **Backend:** Node.js/Express (backend/ folder) with MongoDB, JWT auth, Razorpay payments
- **Frontend:** Static HTML/CSS/JS (frontend/ folder) served by Express
- **Deployment Target:** Render.com (render.yaml already exists)
- **GitHub Repo:** https://github.com/rahulprasad6576/Digital-sewa.git

### Key Files Analysis
- `backend/server.js`: Serves frontend from `../frontend`, requires JWT_SECRET & MONGODB_URI
- `frontend/script.js`: Uses `const API_URL = ""` (relative URLs - correct for same-origin)
- `render.yaml`: Basic config present but missing some env vars
- `backend/seed.js`: Creates default admin user (admin@digitalseva.com / admin123)

### Deployment Requirements
1. **MongoDB Atlas** connection string (MONGODB_URI)
2. **JWT_SECRET** (auto-generated by Render)
3. **Razorpay keys** (optional - demo mode works without them)
4. **CORS origins** must include Render deployment URL

### Step 1: Fix Deployment Configuration
**Files to edit:**

### Step 2: Ensure CORS Compatibility
**Files to edit:**

### Step 3: Commit & Push to GitHub
**Commands:**
- `git add .`
- `git commit -m "Prepare for Render deployment"`
- `git push origin main`

### Step 4: Deploy on Render.com
**Actions:**
1. Go to https://dashboard.render.com
2. Click "New +" → "Blueprint"
3. Connect GitHub repo: `rahulprasad6576/Digital-sewa`
4. Render will auto-detect `render.yaml`
5. Set `MONGODB_URI` environment variable
6. Deploy!

### Step 5: Post-Deployment Setup
**Actions:**
1. Run seed script to create admin user
2. Verify health endpoint: `/health`
3. Test signup/login flow
